Options

The options group box can be used to:

  • Load a previously saved Armageddon options initialization file (*.ini)
  • Save a current Armageddon options initialization file (*.ini)
  • Cmdline opens a dialog which will accept any commandline arguments to be passed to a program prior to execution:

Note: Initialization options must be used explicitly. (i.e. an options (*.ini) file is not loaded when Armageddon is first loaded, nor when a target application file is selected, nor will an options file be saved automatically). It is strongly recommended to use this feature either before opening a target application, or immediately after "refreshing" a session if required. Manually edit an initialization file at your own risk!

This feature can be especially useful when grouping applications with similar option(s) requirements before unpacking. Generic or application specific initialization files can be used as necessary.

Below is the sample template Armageddon options file (armageddon.ini) included in the distribution:

[APPNAME]
armageddon=ARMAGEDDON
[OPTIONS]
cmdline=
redirectcs=0
minimizesize=0
securitydlldump=0
securitydllload=0
openmutex=0
dumppdata=0
ignore2ndtext=0
bypass2ndtext=0
[NANOMITES]
analyzenf=0
analyzest=0
analyzelog=0
[HARDWAREFINGERPRINT]
standard=0000-0000
enhanced=0000-0000

Keyword = (Option value) = 0 means this option is disabled (ignored).
Keyword = (Option value) = 1 means this option is enabled.
(ex: redirectcs=1       Enable the Redirect Code Splicing option).
cmdline=arg1 arg2 arg3 etc...   for any commandline arguments to be passed to a program prior to executing
